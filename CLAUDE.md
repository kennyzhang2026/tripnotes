## 项目概述

**游记助手**是一个基于AI的图文游记生成应用，用户可以拍照/上传照片、添加语音/文字评论，AI自动整理成图文并茂的游记。支持OCR识别照片中的文字（如景区对联）并进行文化解释，保存到飞书多维表格。

### 核心功能
- 批次照片上传（支持多选+拍照）
- 语音输入转文字（阿里云ASR，内嵌按钮）
- OCR识别照片文字（阿里云OCR，自动提取日期）
- AI整体游记生成（DeepSeek，有逻辑的完整游记）
- 保存到飞书多维表格
- 导出分享

### 技术栈
| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | Streamlit | 支持多端访问 |
| AI模型 | DeepSeek API | 游记生成、标题生成 |
| OCR服务 | 阿里云OCR | 500次/月免费（通用文字识别） |
| 图片存储 | 阿里云OSS | 华北2（北京）5GB+5GB流量/月免费 |
| 语音识别 | 阿里云ASR | 高精度语音转文字 |
| 数据存储 | 飞书多维表格 | 用户数据、游记数据 |
| 部署平台 | Streamlit Cloud | 免费部署 |

| ai_content | longtext | AI生成的游记内容 |

## 核心交互流程

### 旧版本流程（v0.2.x）
```
用户进入"创建游记"页面
   ↓
显示已添加的照片+评论列表
   ↓
点击 "+" 按钮展开新输入区
   ↓
拍照/上传照片 → 语音输入 → 文字评论
   ↓
继续添加更多照片+评论
   ↓
点击 "生成游记"
   ↓
上传图片到OSS → OCR识别 → AI生成游记 → 保存到飞书
```

### 新版本流程（v0.3.0 重构）
```
用户进入"创建游记"页面（快速登录，默认kenny）
   ↓
┌─────────────────────────────────┐
│ 批次 1                          │
│ - 点击 + 添加照片（多选/拍照）    │
│ - 在统一输入框填写评论           │
│ - 可点击语音按钮说话转文字        │
│ - 点击"提交这批内容"             │
│ - 图文上传到 OSS                 │
└─────────────────────────────────┘
   ↓
┌─────────────────────────────────┐
│ 批次 2                          │
│ - 继续添加照片                  │
│ - 填写评论                      │
│ - 提交批次                      │
└─────────────────────────────────┘
   ↓
可继续添加更多批次...
   ↓
点击"生成游记"
   ↓
AI 根据所有批次生成完整游记：
- 整合时间线/空间顺序
- 筛选代表性照片
- 融合用户感受
- OCR 文化解释
- 形成有逻辑的整体
   ↓
保存到飞书
```

### 关键交互点
1. **快速登录**：默认填充 kenny 用户，一键进入
2. **+ 照片按钮**：支持多选照片或直接拍照
3. **语音按钮**：内嵌在输入框中，说话转文字直接填入
4. **批次提交**：每批照片+评论独立提交，积累多个批次
5. **整体生成**：AI 生成有逻辑的完整游记，而非机械分块

## v0.3.0 重构技术方案

### 重构背景
当前 v0.2.x 版本的问题：
1. 登录流程繁琐，每次都需要输入密码
2. 每次只能添加一张照片+评论，重复操作多
3. 语音按钮独立在外面，不够直观
4. AI 生成的游记是机械分块，缺乏整体逻辑

### 重构目标

#### 1. 登录优化
- 默认填充用户名 `kenny` 和密码
- 提供"快速登录"按钮，一键进入
- 保留普通登录入口供其他用户

#### 2. 照片输入区域重构
```
┌─────────────────────────────────────────┐
│  📸 照片区域                              │
│  ┌───┐ ┌───┐ ┌───┐                      │
│  │ 1 │ │ 2 │ │ 3 │  已添加的照片预览       │
│  └───┘ └───┘ └───┘                      │
│                                          │
│  [ ➕ 添加照片 ]                          │
│    - 选择已有照片                          │
│    - 打开相机拍照                          │
└─────────────────────────────────────────┘
```

**功能点**：
- 支持批量上传（`st.file_uploader(accept_multiple_files=True)`）
- 支持相机拍照（移动端 `<input accept="image/*" capture="camera">`）
- 照片预览和删除

#### 3. 评论输入区域重构
```
┌─────────────────────────────────────────┐
│  📝 我的感想                              │
│  ┌───────────────────────────────────┐  │
│  │ 在这里记录你的旅行感受...           │  │
│  │                                   │  │
│  │ 提示：可以包含时间、地点、人物等     │  │
│  │ 信息，如：今天下午在西湖边散步...    │  │
│  │                                   │  │
│  │                     [🎤 语音输入] │  │
│  └───────────────────────────────────┘  │
│                                          │
│  [ 📤 提交这批内容 ]                      │
└─────────────────────────────────────────┘
```

**功能点**：
- 统一的文本输入区域
- 智能 placeholder 提示用户输入时间、地点等信息
- 语音按钮内嵌在输入框右下角
- 语音转文字（阿里云 ASR）直接填入输入框

#### 4. 批次提交机制

**数据结构**：
```python
# Session State
st.session_state.current_photos = []      # 当前批次的照片
st.session_state.current_comment = ""     # 当前批次的评论
st.session_state.submitted_batches = []   # 已提交的批次

# 每个批次结构
{
    "batch_id": "uuid",
    "photos": [photo_bytes, ...],
    "comment": "用户评论",
    "timestamp": "提交时间",
    "ocr_results": {...},
    "image_urls": [...]  # 提交后填入
}
```

#### 5. AI 整体游记生成

**新提示词结构**：
```
你是一位专业的游记作家。请根据用户提供的多组照片和评论，写一篇完整的游记。

## 用户素材
- 批次1：照片3张，评论"今天下午来到西湖，断桥残雪..."
- 批次2：照片2张，评论"傍晚时分，雷峰塔的灯光..."
- OCR识别：碑文"西湖十景..."

要求：
1. 写一篇完整的游记，有开头、发展、高潮、结尾
2. 按时间线或空间顺序组织内容
3. 选择有代表性的照片，不必每张都引用
4. 内容相近的照片可合并描述
5. 融入用户评论，保持第一人称视角
6. 对OCR识别的文化内容（如对联、碑文）进行解释
7. 文笔流畅，富有感染力，约 600-1000 字

请以 Markdown 格式输出，照片引用格式：![照片描述](URL)
```

**输出格式**：
```markdown
# 游记标题

今天下午，我来到了心仪已久的西湖...

![断桥远景](照片1_URL)

断桥之上游人如织，我不禁想起白娘子的传说...

根据碑文记载...

傍晚时分...

![雷峰塔夕照](照片4_URL)

这次西湖之行，让我...
```
